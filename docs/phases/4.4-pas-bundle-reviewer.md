# Phase 4.4 — Da Vinci PAS Bundle builder + Reviewer (HITL)

**Week**: 5 · **Status**: ⏳

## Goal

Close the loop: take the Reasoner's Decision, assemble a conformant Da Vinci PAS Bundle, and route through a human-in-the-loop Reviewer agent before any (mock) submission.

## Scope — what ships

1. **PAS Bundle Builder**
   - Deterministic. **No LLM in this node.** The Reasoner's structured Decision contains everything needed.
   - Generates a `Bundle` of type `collection` containing: `Claim` (PAS-profiled), `Patient`, `Coverage`, `Organization` (provider + payer), `Practitioner`, supporting `Observation` / `Condition` / `Procedure` / `DocumentReference` resources by reference.
   - Includes a `Provenance` resource serializing the LangGraph checkpoint hash → full audit trail back to the agent run.

2. **Conformance gate**
   - `$validate` against `us-core` + `davinci-pas` profiles, run in CI.
   - Inferno PAS test kit integration as a stretch goal.
   - **A Bundle that fails validation cannot be emitted from the node.** Architectural enforcement.

3. **Reviewer agent (HITL)**
   - LangGraph `interrupt()` checkpoint — pause graph, persist state.
   - Minimal CLI/web UI: shows the draft Bundle (human-readable view) + structured Decision + citations + source FHIR resource links.
   - Three actions: **approve** → resume to submit · **edit** → modify justification text only, re-validate · **send back** → write structured feedback, replay Reasoner.
   - **Auto-submit is architecturally impossible.** The `submit_to_payer` tool is callable only with a verified Reviewer-approved checkpoint.

4. **Mock submission**
   - Posts the validated Bundle to a local PAS `$submit` endpoint (mock payer server).
   - Returns a mock `ClaimResponse` for the end-to-end demo.

5. **End-to-end demo**
   - `make demo-e2e PATIENT=<id> SERVICE=mri-lumbar-spine`
   - Walks: Intake → Evidence → Reason → Build → Reviewer pause → (human approves in UI) → Submit → ClaimResponse.

## ADRs likely needed

- **ADR-0007** — Reviewer UI surface (CLI vs minimal React vs Streamlit) for v1.0.
- **ADR-0008** — Persistence backend for LangGraph checkpoints (SQLite for dev, Postgres for any shared deployment).
- **ADR-0009** — Mock payer server (custom FastAPI vs HAPI extension).
