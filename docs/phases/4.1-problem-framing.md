**Week**: 2 · **Status**: ▶ In progress

## Goal

Stand up the repo as a credible, leader-grade artifact *before any code is written*. The Phase 4.1 deliverables are the things most open-source repos never ship: problem framing, leadership narrative, ADR discipline, and a public roadmap.

## Definition of Done

- Repo passes the **30-second recruiter scan test**: a healthcare-AI hiring manager opens the repo and within 30 seconds can answer "what does this do, why now, and who is the person behind it?"
- `LEADERSHIP.md` answers "would I trust this person to lead a 4-person squad in a regulated environment?"
- `ADR-0001` demonstrates judgment over framework-name-dropping.

## Why this sub-phase exists

Every other open-source PA project starts with code. This one starts with the **problem, the people, and the decisions** — because the role being targeted (Solutions Architect / Tech Lead) is graded on exactly those three artifacts, not on lines of Python.

## What does NOT happen in 4.1

- No application code.
- No model selection (that's a Phase 4.3 ADR).
- No CI/CD (Phase 4.2, when the first runnable slice lands).
- No demo of the *product* — only of the *problem*.
