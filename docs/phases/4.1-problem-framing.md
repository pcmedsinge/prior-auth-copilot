## Definition of Done

- A first-time visitor can answer "what does this project do and why now?" from the README alone.
- Architecture, personas, and ADR-0001 are in place so Phase 4.2 can begin without re-litigating foundational choices.

## Why this sub-phase exists

Starting with the **problem, the people, and the architectural decisions** — before code — keeps subsequent slices focused and prevents the project from drifting into a demo without grounding.

## What does NOT happen in 4.1

- No application code.
- No model selection (that's a Phase 4.3 ADR).
- No CI/CD (Phase 4.2, when the first runnable slice lands).
- No demo of the *product* — only of the *problem*.
