# Phase 4.2 — Synthea data + FHIR MCP evidence retrieval

**Week**: 3 · **Status**: ✅ Shipped

## Goal

End-to-end vertical slice: given a synthetic patient and a proposed service, retrieve and rank the clinical evidence package via FHIR MCP tools.

## Scope — what ships

1. **Synthea pipeline** ✅ Issue #3 complete
   - Custom Synthea GMF module (`data/synthea-config/modules/low_back_pain.json`) — 4 evidence paths (complete, missing_conservative, short_conservative, nsaids_only).
   - HAPI FHIR v7.4.0 on port 8082 via `docker-compose.yml`; US Core + Da Vinci PAS IGs loaded (ADR-0002).
   - ~50 synthetic patients with varied evidence-completeness loaded via `make load-synthea`.
   - Tag-based idempotent re-load (`meta.tag = synthea-v1`); curated manifest at `data/synthea-config/manifest.json`.
   - Smoke test: `make smoke` verifies patient count, evidence presence, and 5 evidence-missing cases.

2. **FHIR MCP integration** ✅ Issue #4 complete
   - `mcp-fhir` added as dependency (`pyproject.toml`).
   - 6 named evidence tools in `src/prior_auth_copilot/evidence/tools.py`: `find_observations`, `find_conditions`, `find_procedures`, `find_medication_history`, `find_imaging_studies`, `find_documents`.
   - Each wraps `mcp_fhir.tools.fhir_search` with PA-specific parameter defaults.
   - `make mcp-up` starts mcp-fhir in SSE mode on port 8084 (for MCP Inspector / Phase 4.5+).
   - Smoke test: `make smoke-tools` verifies all 6 tools callable against HAPI.
   - `docs/mcp-tools.md` documents each tool's signature and example response.

3. **Evidence ranking** ✅ Issue #5 complete
   - `PAState` TypedDict + Pydantic leaf models in `src/prior_auth_copilot/state.py` (per ADR-0003).
   - Intake stub node validates patient_id + ServiceRequest.
   - Evidence Gatherer node: 6 concurrent tool calls → deterministic PA checklist → gpt-4o-mini one-line summaries.
   - LangGraph graph: Intake → Evidence Gatherer → END.
   - `make demo-evidence PATIENT=<id>` prints structured EvidencePackage + writes JSON.
   - Unit tests for deterministic checklist tagger (no live LLM): `make test`.

4. **Demo** ✅ Issue #7 complete
   - `make demo-evidence PATIENT=<id>` prints structured EvidencePackage to stdout + writes `out/evidence-<id>.json`.
   - Get a patient ID: `python -c "import json; m=json.load(open('data/synthea-config/manifest.json')); print(m['patients'][0]['patient_id'])"`

## ADRs likely needed

- **ADR-0002** — FHIR server choice → ✅ Accepted (HAPI chosen).
- **ADR-0003** — How patient and service context is passed across the LangGraph state (typed TypedDict vs Pydantic model).
