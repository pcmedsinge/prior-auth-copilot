# Phase 4.5 — Evals harness, outcome metrics, v1.0 release

**Week**: 6 · **Status**: ⏳

## Goal

Make the system's quality story public, defensible, and reproducible. Ship v1.0.

## Scope — what ships

1. **Evals harness (formalized)**
   - Promotes the per-phase scorecards into a single `evals/` package.
   - Golden datasets versioned in-repo: `evals/datasets/mri-lumbar-spine-v1.jsonl` (30 cases, clinician-labeled).
   - Runner: `make evals` produces a single Markdown scorecard + JSON metrics.
   - Runs against ≥ 2 model families (e.g., GPT-4-class + Claude-class + an open-weights option) — wired through the ADR-0006 model abstraction.
   - LangSmith / OpenTelemetry traces exported per run.

2. **Outcome metrics dashboard**
   - Published as `docs/evals/v1.0-scorecard.md` and linked from README hero block.
   - Headline metrics: citation grounding · criterion accuracy · Bundle `$validate` pass rate · P50/P95 latency · cost per PA draft.
   - **Each metric reports the failure case**, not just the headline number. Hide nothing.

3. **`docs/runbook/`**
   - On-call escalation, common failure modes, how to inspect a stuck LangGraph checkpoint, how to roll back an embedding-model change.

4. **v1.0 release**
   - Tagged release `v1.0.0` with release notes covering: scope, eval scorecard, known limitations, what's next.
   - GitHub Release published.
   - LinkedIn long-form post (1000+ words): *"Building an open-source agentic Prior-Auth co-pilot against the CMS-0057 deadline — what worked, what didn't, what I'd staff differently."*
   - Asks made: contributors for additional payer policies, design partners for real-world case validation.

5. **Pin the repo** as #1 in the GitHub profile pinned order. Update profile README pinned section.

## ADRs likely needed

- **ADR-0010** — Eval dataset versioning and contribution model (how external clinicians can submit cases without exposing PHI).
- **ADR-0011** — Release / versioning policy (semver interpretation for an agentic system — what counts as a breaking change?).

## Definition of Done

- `make evals` runs clean and produces the v1.0 scorecard from a fresh checkout.
- README hero block shows the v1.0 scorecard numbers.
- v1.0.0 GitHub Release tagged and published.
- LinkedIn long-form post published; reshared in HL7, FHIR, LangGraph, MCP communities.
- Repo pinned in #1 position on GitHub profile.
- Phase 4 entry in workspace `PLAN.md` marked ✅ Complete.
