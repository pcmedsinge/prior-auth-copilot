# User personas

Three personas drive every design decision. If a feature doesn't serve at least one of them, it doesn't ship.

- [Ordering provider](ordering-provider.md) — wants explainable PA drafts, not another portal.
- [Utilization management (UM) nurse](um-nurse.md) — needs evidence-traceable, edit-before-submit workflow.
- [Payer reviewer](payer-reviewer.md) — receives clean, standards-compliant PAS Bundles with citations.

Each persona doc is structured as: *Day in the life → What they need from this tool → What would make them stop using it.*
